# Финальный проект по курсу «Цифровая архивация современности»

Репозиторий посвящён коллекции технических документаций веб-технологий и фреймворков, собранной при помощи [wpull](https://github.com/ArchiveTeam/wpull). Здесь опубликованы метаданные каждого элемента коллекции, созданные с использованием инструмента [metawarc](https://github.com/datacoon/metawarc), базы данных, а также результаты анализа сайтов с помощью сервиса [ArchiveReady](https://archiveready.com/), определяющего индекс и показатели архивируемости сайтов.

## Краткое описание коллекции

В коллекцию вошли сайты технических документаций популярных веб-технологий, языков программирования и инструментов:

1. [Alpine.js Documentation](https://alpinejs.dev/start-here)
2. [cURL Documentation](https://curl.se/docs/)
3. [The Rust Programming Language Book](https://doc.rust-lang.org/book/)
4. [Nginx Documentation](https://nginx.org/en/docs/)
5. [Lua 5.4 Reference Manual](https://www.lua.org/manual/5.4/)

Вся коллекция содержит файлы разных форматов, которые необходимы для функционирования сайтов документации.

Важность сохранения этих сайтов заключается в необходимости создания долговременных архивов технической документации, которая может быть изменена, удалена или стать недоступной со временем. Эти архивы обеспечивают доступ к конкретным версиям документации, что критически важно для поддержки legacy-проектов и исследования эволюции технологий.

## Метод и технологии

При сборе и обработке коллекции применялись следующие инструменты:

1. Библиотека **wpull**, доступная на Python: предназначена для создания локальных WARC-архивов сайтов с возможностью настройки параметров глубины рекурсии, обработки robots.txt и других параметров архивирования.
2. Утилита командной строки **metawarc**, позволяющая извлекать метаданные из WARC-файлов, создавать базы данных и анализировать содержимое архивов.
3. Сервис **ArchiveReady**, определяющий индекс архивируемости сайтов и оценивающий их по четырём категориям: Accessibility (доступность), Cohesion (связность), Metadata (метаданные) и Standards Compliance (соответствие стандартам) в соответствии с методикой [CLEAR](https://purl.pt/24107/1/iPres2013_PDF/CLEAR%20a%20credible%20method%20to%20evaluate%20website%20archivability.pdf).

## Структура проекта

1. Отдельные **папки**, названные в соответствии с доменными именами архивированных сайтов. Внутри каждой папки:
   - **README.md** – описание сайта, результаты анализа ArchiveReady и обработки архива с помощью metawarc, сбор файла автоматизирован на Python
   - **[название_сайта].warc.gz** – сжатый WARC-архив сайта
   - **[название_сайта].warc_metadata.jsonl** – метаданные архива в формате JSON Lines
   - **archive_ready_[название_сайта].json** – результаты анализа архивируемости от сервиса ArchiveReady
   - **[название_сайта].log** – лог процесса архивирования wpull
   - **[название_сайта].cdx** – CDX-индекс архива
   - **metawarc.db** – база данных, созданная на основе metawarc
   - **sitearchive-[название_сайта].db** – файл базы данных, созданный при использовании wpull 

2. **warc_archive_report.md** – сводный отчёт по всем архивам коллекции
3. **source_code.ipynb** - файл формата jupyter-notebook, с помощью которого производилась архивация и генерация .md файлов
4. **README.md** – описание проекта
